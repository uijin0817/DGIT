📁

프로젝트 시작하기

홈 화면에서 '새 프로젝트 선택'을 눌러 관리하고 싶은 폴더를 선택하세요.
DGit 저장소가 없는 폴더는 자동으로 초기화할 수 있습니다.

💾

변경사항 커밋하기

파일을 수정하면 '프로젝트 파일' 목록에서 상태가 'modified'로 바뀝니다.
'변경사항 커밋' 버튼을 눌러 변경된 파일을 저장하세요.

↩️

이전 버전으로 복원

'커밋 히스토리' 탭에서 특정 커밋을 선택하고 '복원' 버튼을 누르면 해당 시점으로 돌아갈 수 있습니다.

⚠️

복원 및 초기화 주의사항

'복원' 기능은 현재 프로젝트의 모든 변경사항을 마지막 커밋 상태로 되돌립니다. 이 과정에서 커밋되지 않은 파일 변경 내용은 **영구적으로 삭제**됩니다. 신중하게 사용하세요.

DGit 저장소 초기화는 숨겨진 버전 관리 폴더를 생성합니다. 프로젝트에 민감한 파일을 포함하는 경우, Gitignore 파일을 수동으로 편집하여 관리 대상에서 제외할 수 있습니다.

🟢

파일 상태 표시 설명

Committed / Added: 저장소에 기록되거나 스테이징된 파일.

Modified: 파일이 수정되었지만 아직 커밋되지 않은 상태.

Untracked: 새로 생성되어 DGit이 아직 추적하지 않는 파일. '모든 파일 추가' 필요.

Deleted: 파일이 삭제되었거나 제거된 상태.

⌨️

키보드 단축키 (macOS)

⌘ + N

새로운 DGit 프로젝트 폴더를 선택하고 엽니다.

⌘ + O

가장 최근에 작업한 프로젝트를 다시 로드합니다.

⌘ + S

모든 변경사항을 스테이징하고 커밋 모달을 엽니다 (DGit Commit).

⌘ + W

현재 워크스페이스를 닫고 홈 화면으로 돌아갑니다.

⌘ + R

현재 프로젝트의 파일 및 상태 정보를 새로고침합니다.

⌘ + T

터미널 패널을 확장하거나 축소합니다.

Esc

열려 있는 모달 창을 닫거나, 모달이 없으면 홈 화면으로 돌아갑니다.

⚡

DGit 명령어 소개

dgit init

새로운 DGit 저장소를 현재 디렉터리에 생성합니다.

dgit status

현재 프로젝트의 변경된 파일 상태를 확인합니다.

dgit add [파일/폴더]

변경사항을 스테이징 영역에 추가합니다.

dgit commit -m "[메시지]"

스테이징된 파일을 커밋하여 저장소에 기록합니다.

dgit log

커밋 히스토리를 시간 순서대로 보여줍니다.
